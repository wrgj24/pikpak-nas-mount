## 把 PikPak 挂载到 NAS 上：三种方案对比

这是 PikPak NAS 玩法里最实用的部分。目前主流有三种方式把 PikPak 挂到群晖（Synology）或极空间上：

### 方案一：官方 WebDAV + Cloud Sync（需要 Premium 会员）

PikPak 从 2023 年起提供官方 WebDAV 服务，地址是 `http://dav.mypikpak.com`。配合群晖 DSM 内置的 **Cloud Sync** 功能，可以把 PikPak 网盘当作一个外部云存储定期同步到本地。

**操作步骤大致如下：**

1. 打开群晖 DSM → 套件中心 → 安装 **Cloud Sync**
2. 打开 Cloud Sync，点击 `+` 号，选择 **WebDAV**
3. 填入 URL：`http://dav.mypikpak.com`，填入 PikPak 账号密码
4. 设置同步目录和方向（建议单向：云端 → 本地）
5. 等待同步完成即可

**优点**：配置简单，稳定，适合定期同步归档。
**缺点**：必须 Premium 会员才能使用官方 WebDAV，且 Cloud Sync 是定时同步而非实时挂载，不适合"直接播放 PikPak 文件"的场景。

---

### 方案二：rclone + SSH（免会员可用，功能最强）

rclone 官方已经原生支持 PikPak，可以直接调用 PikPak 的 API，无需 Premium 会员也能挂载。这也是目前 NAS 玩家最推荐的方案。

**操作步骤：**

1. SSH 进入群晖（需提前在控制面板开启 SSH）
2. 安装 rclone：
   bash
   curl https://rclone.org/install.sh | sudo bash
   
3. 配置 PikPak 远程源：
   bash
   rclone config
   
   按提示选择 `n`（新建），命名为 `my-pikpak`，存储类型选 **PikPak**，输入账号密码完成配置。

4. 验证是否连接成功：
   bash
   rclone ls my-pikpak:'My Pack'
   

5. 启动 WebDAV 服务，供群晖 File Station 或 Emby 访问：
   bash
   rclone serve webdav \
     --addr 0.0.0.0:8000 \
     --cache-dir /volume1/.cache/rclone \
     --vfs-cache-mode full \
     my-pikpak:'My Pack'
   

6. 在 File Station 中新增远程挂载，地址填 `http://127.0.0.1:8000` 即可。

**优点**：免会员可用，访问体验更接近本地网络驱动器，可以配合 Emby / Plex 直接扫描 PikPak 库。  
**缺点**：配置稍复杂，需要通过 SSH 操作；建议设置 rclone 开机自启脚本，否则重启后需要手动执行。

---

### 方案三：Alist / CloudsLinker（Docker 一键部署）

如果你不想折腾命令行，**Alist** 是一个开源的多云存储聚合工具，支持通过 Docker 部署，配置界面友好，同样支持 PikPak。

**大体流程：**

1. 在群晖 Container Manager（原 Docker）中搜索 `alist` 镜像并运行
2. 访问 `http://NAS-IP:5244` 打开 Alist 管理界面
3. 添加存储，选择 PikPak，填入账号信息
4. Alist 会生成一个 WebDAV / HTTP 地址，直接挂载到 File Station 或 Infuse 等播放器

**优点**：图形化操作，对新手友好，支持多账号、多云盘聚合。  
**缺点**：Alist 是第三方工具，账号密码需要填入其服务，介意安全性的用户要酌情评估。
